Use Case

For this process, I created a use case. I chose a use case because there are multiple things someeone can do through my website. I created 4 basic use cases. This use case applies to one audience/actor, which is post graduate designers. The four use cases was the post grad designer wants to have their portfolio critiqued, wants to critique other designer's work, know how the website works, and wants to know how to critique properly. They all had straight forward and linear descriptions except for the second one, which is that the post grad designer wants to critique other designer's work. The second alternative is going through the community page first rather than logging on first.
